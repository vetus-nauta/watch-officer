# Watch Officer — Стартовый протокол дня

Используй этот шаблон для каждого нового дня.

1. Проверка контрольной позиции
1. Уточнить текущий `checkpoint` и последнее утвержденное `task-registry` состояние.
1. Проверить `git status --short`.
1. Проверить синхронизацию: `git rev-list --left-right --count HEAD...origin/main` должно быть `0 0` (или зафиксировать отклонения).
1. Если есть отклонения, зафиксировать их в заметках и решить первоочередной источник.

2. Назначение ежедневного фокуса
1. Сформулировать один главный приоритет в формате `TASK-xxxx start`.
1. Записать цель одной строкой и ожидаемый следующий статут.
1. Запретить сразу 3+ независимых потоков: только один основной поток, остальные после `qa ok`/`deploy ok` предыдущего шага.

3. Ограничение изменений на день
1. Открыть только явно затронутый поток задач.
1. Не трогать Captain Ether, Nav Desk, router/registry, auth, production config без отдельного назначения.
1. Не запускать новые широкие проверки без запроса.
1. Доказывать изменения короткими отчётами и явно указанным `scope`.

4. Шаблон ежедневного запуска
- `TASK-xxxx start. Главный приоритет: ...`
- `Scope: ...`
- `Expected next: ...`
- `Risk/Blockers: ...`
- `Decision needed: ...`

5. Правило закрытия дня
1. Если день закрылся milestone — сделать checkpoint (tag + note).
1. Указать статус: что закрыто, что перенесено, что требует подтверждения.
1. Перечень незавершенных задач оставить в одном блоке `Next steps`.

Сохранять этот порядок до явного изменения процесса.
