# Product Roadmap

## Phase 0 - Project Office

Create the office structure, decision process, reports, templates, and central indexes.

## Phase 1 - Product Bible

Approve vision, scope, target users, vocabulary, MVP definition, and roadmap.

## Phase 2 - Gameplay Architecture

Define scenario schema, navigation rules, traffic rules, VHF event format, and evaluation model.

## Phase 3 - Technical Prototype

Build a tiny 2D prototype after approval.

## Phase 4 - Buoy Logic Prototype

Add core marks, shallow zones, and grounding detection.

## Phase 5 - Encounter Prototype

Add one traffic target and simple CPA/TCPA warning.

## Phase 6 - Scenario Engine

Add scenario resources, checkpoints, objectives, fail states, scoring, and evaluation.

## Phase 7 - VHF Prototype

Add popup dialogue, response input, validation, and correction remarks.

## Phase 8 - MVP Polish

Add 5-10 short scenarios, consistent UI, evaluation screen, and mobile usability pass.

## Later Branches

- Night/AIS branch.
- Sailing branch.
- More advanced traffic separation and restricted visibility.
