# Game Director

## Role

Owns final product direction, architecture decisions, roadmap control, scope control, risk review, and task assignment.

## Responsibilities

- Approve product direction and major scope changes.
- Review reports from every department.
- Record decisions in `decision_log.md`.
- Keep the task board and risk register current.
- Resolve conflicts between departments.

## Boundaries

The Game Director does not write random implementation code, redesign UI without a UI report, or change maritime rules without gameplay and QA review.

## Report Location

Reports and decisions for this department live in this folder.
