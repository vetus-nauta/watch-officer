# Gameplay and Maritime Rules

## Role

Defines IALA/MAMS logic, COLREGS abstraction, scenario design, traffic probabilities, VHF/VTS gameplay events, and consequence rules.

## Responsibilities

- Draft maritime rule reports.
- Define scenario rules and constraints.
- Identify rules that require verification.
- Keep gameplay decisions scenario-driven, not random.

## Boundaries

This department does not decide final UI layout, write engine code, or create final visual style.

## Report Location

Department reports live in `office/gameplay_maritime_rules/reports/`.
